2. Security Layers

Every connection passes through multiple security layers. No direct access to client systems is ever possible.

🔒
TLS 1.3 Encryption
All traffic encrypted in transit. No plaintext data ever crosses the network. This is the same encryption used by banks.
🛡️
Cloudflare WAF
Web Application Firewall blocks SQL injection, XSS, and other attacks before they reach our Workers. Same protection used by 20% of the internet.
🔑
OAuth 2.0 Authentication
We never see or store passwords. Clients authorise specific access (e.g. "read my Gmail") via Google/Microsoft OAuth. They can revoke access anytime.
🗄️
Secrets Encryption at Rest
API keys and OAuth tokens stored as Cloudflare Secrets — encrypted at rest, never visible in code, never in logs, only accessible by the Worker at runtime.
🚇
Cloudflare Tunnel (Zero Trust)
No open ports. No VPN. No firewall holes. The tunnel agent initiates an outbound-only encrypted connection. Nothing from the internet can reach the client's network directly.
⏱️
Rate Limiting
Built-in rate limiting prevents abuse. Maximum 10 audits/hour, API calls throttled, no unlimited access to any endpoint.
🧠
AI Data Handling
Claude (Anthropic) processes data but does NOT store it. No training on customer data. Each request is stateless — data is processed and forgotten. Anthropic's privacy policy guarantees this.
📧
Email via Resend
Transactional email only — no marketing lists, no data sharing. Resend is SOC 2 compliant. Emails sent, delivered, logged for 30 days, then purged.

5. Attack Surface Analysis

What could an attacker try, and how it's prevented.

Attack: Intercept data between client and Cloudflare
Mitigated: TLS 1.3 encryption on all connections. No plaintext ever transmitted. Certificate pinning on tunnel.
Attack: Hack into client's network via the tunnel
Mitigated: Tunnel is outbound-only. No ports are opened. No inbound connections possible. The cloudflared agent controls exactly which services are exposed.
Attack: Steal API keys / OAuth tokens
Mitigated: Stored as Cloudflare Secrets — encrypted at rest, never in code, never in logs, only decrypted in Worker memory at runtime.
Attack: Access customer data via AI provider
Mitigated: Anthropic (Claude) does not store or train on data from API calls. Each request is stateless. No customer data persists at Anthropic.
Attack: DDoS the audit endpoint
Mitigated: Cloudflare's built-in DDoS protection + our rate limiter (10 requests/hour). Cloudflare absorbs attack traffic before it reaches Workers.
Attack: Man-in-the-middle on email
Mitigated: Resend uses TLS for delivery. DKIM/SPF/DMARC configured on layerops.tech domain. Email content is transactional, not sensitive data.
Attack: Compromise a Worker to access other clients' data
Mitigated: Each Worker runs in an isolated V8 sandbox. Workers cannot access each other's memory or secrets. KV data is namespaced per purpose.

6. Compliance & Data Handling

Australian Privacy Act
All data processed via Cloudflare's Australian/US edge nodes. Customer data handled per APP guidelines. No offshore storage of PII beyond encrypted transit.
Data Minimisation
We only access what's needed. Gmail triage reads subjects and previews — not full email bodies unless required. OAuth scopes are minimal.
Client Control
Clients can revoke OAuth access anytime via Google/Microsoft settings. Tunnel can be stopped by unplugging the agent. No lock-in.
Data Retention
Lead data: 90 day TTL. Chat conversations: not stored. Email content: processed and discarded. Audit results: 90 days. Client configs: until they cancel.
